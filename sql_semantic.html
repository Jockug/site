Семантика SQL

Для начала, SQL - универсальный компьютерный язык, применяемый для создания, модификации и управления данными в реляционных базах данных. SQL основывается на исчислении кортежей.
Семантика - наука о понимании определённых знаков, последовательностей символов и других условных обозначений.

Виды семантики:

  • декларативная - описывает знаки и значения в предметной области
  • проективная - если есть другая модель, то в качестве знаний могут предполагаться другие модели
  • рефлексивная - знаки моделей с частями моделей

Мы разобрались с понятием SQL и семантики. Теперь перейдем к SQL-инйкциям. Существует 5 основных видлв инйекций.

  1. Классическая: Самая опасная сегодня атака, но редко встречающаяся. Позволяет сразу получать данные из базы данных.
  2. Error-based: Позволит получать информацию о базе, таблицах, и данных на основе выводимого текста ошибки СУБД1.
  3. Boolean-based: Вместо получения всех данных, атакующий может поштучно их перебирать, ориентируясь на простой ответ типа true/false.
  4. Time-based: Похожа на прошлую атаку методом перебора, манипулируя временем отклика базы данных.
  5. Out-of-Base: Очень редкие и специфические атаки, основанные на индивидуальностях особенностях базы данных.

    6. Все уязвимости находятся в местах где формируется запрос к базе данных.
